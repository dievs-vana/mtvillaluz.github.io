Disbursement Management System (DMS)

Finance events → templated, traceable email

  • Transactional templates for disbursement-related events (aligned with workflow states in the core DMS).
  • Microsoft SMTP Relay for outbound delivery and operational reliability.
  • Recipient targeting from workflow metadata — not static mailing lists.
  • Auditable send path: PHP mail + SMTP config suitable for finance and compliance expectations.

Customer Portal

External-facing messages with consistent branding

  • Billing & statement-oriented message types (aligned with portal data, not generic blasts).
  • Activity and status emails tied to account events customers actually care about.
  • Same SMTP pipeline as internal systems — one configuration story for ops.
  • No secrets in the UI: credentials and relay settings live server-side only.

Technical implementation

  • Email infrastructure: Microsoft SMTP Relay — high-reliability delivery for transactional system notifications.
  • Trigger-based design: Event-driven notifications (approvals, updates, submissions) fired from real domain events — not cron-only noise.
  • Backend integration: PHP mail handling with SMTP configuration, aligned with production security and audit expectations.
  • Scalability: Same notification patterns reused across multiple product surfaces and deployments without duplicating logic everywhere.
